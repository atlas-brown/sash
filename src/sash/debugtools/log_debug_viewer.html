Overview

This viewer helps inspect symbolic execution logs: source coverage by states, pass-aware filtering, a state graph with ancestry, and per-state details.

Loading Logs
  • Click Load Log File and select a .jsonl log generated by the debugger.
  • If the log contains a source_file record, the source pane will populate automatically.
Pass Filtering
  • Use Filter by Pass to show only states, assertions, and reports from a specific analysis pass.
  • Graph nodes are labeled with the passes that reached them; filtering dims nodes outside the selected pass.
Graph Interactions
  • Click a node to select a state. The corresponding source line is highlighted, and ancestor lines show a 🧬 icon in the gutter.
  • Click an edge to view a concise diff between the parent and child states (Added / Removed / Changed).
  • Use marquee or Ctrl/Cmd+Click for multi-select (graph-only; source highlighting follows single-node selection).
Source Pane
  • Line badges: N active states, reports, assertions.
  • Click a line to select all visible states at that line (clears 🧬 markers since there are multiple states).
  • Click the gutter to toggle a details drawer with states, reports (by pass), and assertion summaries.
Details Pane
  • Shows the selected state’s fields and metadata. The _meta section appears last and includes error/report/parent info.
  • When an edge is selected, shows a color-coded diff for Added / Removed / Changed fields.
Tip: Use the pass filter to focus the graph and source on a single analysis pass.
